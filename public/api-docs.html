Invexa API

API RESTful para integração com o sistema de gestão Invexa. Todos os dados são isolados por empresa (multi-tenant).

Base URL
https://invexa-app.com.br/api/v1

Todas as requisições devem incluir o header Accept: application/json. Respostas sempre em JSON.

🔐 Autenticação

A API usa Bearer Tokens gerados via Sanctum. Inclua em todas as requisições protegidas:

Authorization: Bearer {seu_token}
💡 Gere seu token em Invexa → Settings → API Tokens ou via endpoint POST /auth/token.

⚠️ Erros

CódigoSignificado
200Sucesso
201Criado com sucesso
401Não autenticado — token inválido ou ausente
403Sem permissão — recurso de outra empresa ou limite de plano
404Recurso não encontrado
422Erro de validação — ver campo errors
429Rate limit excedido

⏱ Rate Limiting

60 requisições por minuto por token. Headers de resposta indicam o status:

X-RateLimit-Limit: 60
X-RateLimit-Remaining: 58

Autenticação

POST /auth/token Gerar token de acesso

Gera um token Bearer para autenticação. Não requer token existente.

Body (JSON)

CampoTipoObrigat.Descrição
emailstringsimE-mail do usuário
passwordstringsimSenha do usuário
device_namestringsimNome do dispositivo/integração
Exemplo de requisição
curl -s -X POST https://invexa-app.com.br/api/v1/auth/token \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -d '{
    "email": "admin@suaempresa.com",
    "password": "sua_senha",
    "device_name": "minha-integracao"
  }'
Resposta 201
{
  "token": "1|abc123...",
  "token_type": "Bearer"
}
GET /me 🔒 Auth

Retorna dados do usuário autenticado e sua empresa.

Resposta 200
{
  "user":    { "id": 1, "name": "Administrador", "email": "admin@empresa.com", "role": "admin" },
  "company": { "id": 1, "name": "Empresa Demo", "plan": "pro" }
}
DEL /auth/token Revogar token atual
Resposta 200
{ "message": "Token revogado com sucesso." }

Produtos

GET /products Listar produtos

Query params

ParamTipoDescrição
searchstringBusca por nome
activebooleanFiltrar por status (true/false)
per_pageintegerItens por página (padrão: 20)
pageintegerPágina atual
Exemplo
GET /api/v1/products?search=camiseta&active=true&per_page=10
Resposta 200 (paginada)

  "current_page": 1,
  "data": [

      "id": 5,

      "sku": "SKU001",

      "cost": "18.00",
      "quantity": 45,
      "min_stock": 5,
      "active": true

  "per_page": 20,
  "total": 1

Estoque

GET /stock Listar estoque

Retorna todos os produtos com suas quantidades em estoque.

Resposta 200
{
  "data": [
    { "id": 5, "name": "Camiseta Básica", "sku": "SKU001", "quantity": 45, "min_quantity": 5 }
  ]
}
GET /stock/low Produtos com estoque baixo

Retorna produtos onde quantity ≤ min_quantity.
